Highlights

from 2.0

  • QUERY_PLAN Support - Added option -Dbqt.query.plan=true, when using Teiid as the data source, will generate or compare the query plan created by the Teiid engine based on the query executed.
  • METADATA_FILTER options - Added metadata filter options to be used when reading the databasemetadata when resultMode of SQL is used. See the test.properties for the filter options.

from 1.0

  • The initial release is making the BQT tool an open source project.
  • The tool supports the executing queries using the follow resultModes:
    • SQL - create SQL query file for all tables defined in databasemetadata
    • GENERATE - generate expected results files for each SQL query executed
    • COMPARE - compare the actual results to the expected results for each executed query and provide results
    • NONE - execute queries and report which queries fails
  • Muti Scenario Support - supports running 1 or more scenarios in a single execution so that different parameters can be tested against the same source
